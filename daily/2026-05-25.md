## 🗺️ AI × Web3 问题地图

今天整理了 6 个方向：

| 方向 | 我的理解 |
| --- | --- |
| Payment / Commerce / Settlement | AI 可以解释支付流程，Web3 提供真实结算记录 |
| Identity / Reputation / Capability | AI 可以整理能力声明，Web3 提供可验证行为历史 |
| Wallet / Permission / Safe Execution | AI 可以辅助操作说明，Web3 负责权限边界和安全执行 |
| Privacy / Security / Sovereignty | AI 可以做检查和提醒，但不能代替安全设计 |
| Dev Tooling / Agent Workflow | AI 把链上底层数据变成开发者和学习者能读懂的内容 |
| Governance / Coordination / Public Goods | AI 辅助协作和记录，Web3 提供公开透明的 proof |

最后我在两个方向里做了选择：

- `Wallet / Permission / Safe Execution`
- `Dev Tooling / Agent Workflow`

最终主线选择了后者，因为它最贴近我已经做出来的 demo，也最容易继续迭代。

## 💡 今天学到的点

- Week 2 不是马上继续做功能，而是先判断方向是否真的有 AI × Web3 的交叉价值。
- 纯 AI 容易编造链上事实，所以必须依赖真实 transaction、receipt、logs 和 explorer。
- 纯 Web3 数据太底层，AI 可以把它解释成学习者和开发者能读懂的 proof。
- 只读型 agent 是比较安全的第一步，比直接让 agent 操作钱包更适合当前阶段。
- 我的主线应该围绕“链上数据解释 + proof 生成 + 安全边界”继续往下拆。
