## Safety Boundaries

The Agent should not:

- See or store private keys.
- See or store mnemonics.
- Ask me to paste secrets into chat.
- Create public `.env` files or API/private-key placeholders without explicit review.
- Auto-sign wallet messages.
- Auto-send transactions.
- Auto-approve token allowances.
- Auto-confirm contract writes.
- Push public changes without my confirmation.

The Agent may help:

- Draft notes.
- Generate checklists.
- Explain concepts.
- Prepare commands.
- Review public transaction data.
- Organize proof-of-work.
- Compare wallet prompts with expected actions.
- Identify what must be manually verified.
- Suggest safer testnet-first workflows.

## Reflection

The useful pattern is not "let AI do everything." The useful pattern is:

AI drafts -> I review -> I approve sensitive actions -> I verify output -> I record proof.

中文总结：

AI 可以帮助我整理学习路径、解释概念、生成记录和检查公开链上数据，但涉及钱包、签名、转账、授权和合约写入时，必须由我人工确认。学习 Agent 的价值不是替我接管 Web3 操作，而是帮助我把学习过程变成可验证、可复盘、可提交的 proof-of-work。

English summary:

AI can help me organize the learning path, explain concepts, generate records, and check public on-chain data. But wallet actions, signatures, transfers, approvals, and contract writes must be manually confirmed by me. The value of the learning Agent is not to take over Web3 operations, but to help turn my learning process into verifiable, reviewable, and publishable proof-of-work.
