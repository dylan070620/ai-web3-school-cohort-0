# Handbook Feedback Workflow

Use this folder to turn learning friction into useful feedback for the AI x Web3 School Handbook.

## When To Create Feedback

Create a note when:

- A concept is unclear after reading and trying it.
- A setup step is missing, outdated, or hard to reproduce.
- A command, dependency, or link does not work.
- A beginner would need an example, diagram, glossary, or warning.
- A privacy or security risk should be made more explicit.

## Feedback Note Format

Use one file per feedback item:

`YYYY-MM-DD-topic.md`

Include:

- Handbook section or URL.
- What I expected.
- What happened.
- Why it matters for learners.
- Suggested fix.
- Evidence, if safe to share.

## Status Labels

- `draft`: still clarifying the issue.
- `ready`: clear enough to submit.
- `submitted`: sent to the maintainers or community.
- `closed`: resolved or no longer relevant.

## Privacy Rules

- Do not include private keys, API keys, wallet backups, or sensitive personal data.
- Redact wallet addresses if needed.
- Avoid screenshots that reveal browser profiles, tokens, or private tabs.
